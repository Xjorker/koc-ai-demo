### 二、需求背景

#### 2.1 市场痛点

| 痛点 | 描述 | 影响 |
|------|------|------|
| **内容定位模糊** | 不知道自己适合做什么方向，难以形成独特风格 | 内容散乱，粉丝粘性低 |
| **选题低效** | 花费大量时间思考拍什么，选题靠碰运气 | 创作效率低，更新不稳定 |
| **运营不专业** | 不懂平台算法、发布时间、互动技巧 | 流量获取困难 |
| **涨粉困难** | 缺乏系统性的成长路径和策略 | 账号发展缓慢 |

#### 2.2 目标用户

- **初级KOC**：粉丝数 < 1万，内容创作经验不足
- **转型KOC**：从其他行业转向社交媒体，需要快速上手
- **副业KOC**：利用业余时间运营账号，需要高效工具

### 三、产品介绍

#### 3.1 产品定位

**KOC AI增长助手** 是一款面向普通KOC的社媒AI Agent，通过AI能力提供 **内容方向、选题建议、发布策略、互动优化、AI小游戏** 等服务，帮助其低成本、高效率实现账号成长。

#### 3.2 核心价值主张

> "让每个普通人都能轻松玩转社交媒体"

- **AI驱动**：集成大语言模型，提供智能化服务
- **一站式**：覆盖账号运营全流程
- **低成本**：无需专业团队，个人即可使用
- **高效率**：AI辅助决策，大幅提升运营效率

#### 4.2 功能详解

##### 4.2.1 AI账号诊断

**功能描述**：用户输入账号名称和简介后，AI深度分析账号定位与内容策略

**输出内容**：
1. **账号定位分析** - 当前定位评估、优势分析、改进建议
2. **内容策略规划** - 内容矩阵建议（核心/互动/轻松内容配比）、发布频率、最佳发布时间
3. **受众画像分析** - 核心受众特征、受众需求洞察、兴趣标签
4. **30天涨粉计划** - 每周具体行动项、关键里程碑、目标指标
5. **账号健康度评分** - 综合评分（0-100）、各维度雷达图

##### 4.2.2 智能选题引擎

**输入参数**：
- 内容赛道：生活方式、科技数码、美食探店、旅行户外、时尚美妆、健身运动、知识教育、娱乐八卦
- 创意程度：稳健实用、平衡创新、大胆创新
- 选题数量：3-10个

**输出内容**：
- 选题标题 + 描述
- 难度等级（简单/中等/挑战）
- 相关标签 + 今日热点结合

##### 4.2.3 内容创作助手

| 工具 | 输入 | 输出 |
|------|------|------|
| **视频脚本生成** | 视频主题、时长、风格 | 完整分镜脚本（开头吸睛→核心价值→互动引导→结尾升华） |
| **标题优化** | 内容主题、目标平台 | 5-10个备选标题 + 吸引力评分 |
| **文案生成** | 内容主题 | 完整社交媒体文案 |
| **封面建议** | 视频主题 | 封面设计建议（视觉/文字/色彩/元素） |

##### 4.2.4 AI小游戏生成 ⭐（赛道核心功能）

**功能描述**：基于游戏主题和类型，AI一键生成完整的小游戏设计方案，包含规则、分享机制、视觉素材建议。

**输入参数**：
| 参数 | 选项 |
|------|------|
| 游戏主题 | 用户自定义（如：测测你的反应速度） |
| 游戏类型 | 挑战类、知识问答类、互动测试类、抽奖类 |
| 时长要求 | 5-30秒 |
| 风格 | 欢乐有趣、酷炫潮流、可爱清新 |

**输出内容**（JSON格式）：
```json
{
    "title": "游戏标题（吸引眼球）",
    "description": "一句话描述",
    "rules": ["规则1", "规则2", "规则3"],
    "scoring": "得分机制描述",
    "share_text": "分享文案，可嵌入{{score}}、{{result}}、{{prize}}",
    "virality_tips": ["传播技巧1", "传播技巧2"],
    "assets": ["素材描述1", "素材描述2"],
    "game_flow": "游戏流程简述"
}
```

**技术实现**：
- 后端：`content_agent.py` 中 `generate_game()` 函数
- 前端：调用 `/api/game` 端点，接收AI返回的JSON游戏设计
- AI提示词：包含完整游戏设计规范，引导AI输出结构化JSON

**应用场景**：
- 品牌营销：结合品牌元素生成互动小游戏
- 内容引流：游戏结果分享实现病毒式传播
- 粉丝互动：提升粉丝活跃度和粘性

##### 4.2.5 数据看板

**展示内容**：
1. **核心指标卡片** - 粉丝数、播放量、点赞数、评论数（含周增长）
2. **趋势图表** - 粉丝增长趋势图、内容表现分析图
3. **AI数据洞察** - 内容效果分析、最佳发布时间建议、互动率优化建议
4. **内容发布日历** - 已发布内容标记、计划发布日程

##### 4.2.6 成长路径规划

**AI提供**：账号阶段评估 → 下一阶段目标设定 → 具体行动计划 → 资源推荐

### 五、技术架构

#### 5.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                        前端（Web单页应用）                      │
│              HTML5 + CSS3 + Vanilla JavaScript               │
│                      localhost:8080                          │
└─────────────────────────────────────────────────────────────┘
                              ↓ HTTP
┌─────────────────────────────────────────────────────────────┐
│                      后端（FastAPI + LangGraph）               │
│                       localhost:8000                          │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐          │
│  │ diagnosis  │  │   topic    │  │  content    │          │
│  │   _agent   │  │   _agent   │  │   _agent    │          │
│  └─────────────┘  └─────────────┘  └─────────────┘          │
│                         ↓                                    │
│              ┌──────────────────────┐                       │
│              │    AI Client         │                       │
│              │  (OpenAI兼容API)     │                       │
│              └──────────────────────┘                       │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                   外部AI服务（OpenAI/混元/DeepSeek）           │
└─────────────────────────────────────────────────────────────┘
```

#### 5.2 前端技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **页面结构** | HTML5 | 语义化标签，单页应用 |
| **样式设计** | CSS3 | 渐变色、模糊效果、响应式布局 |
| **交互逻辑** | Vanilla JavaScript | 无框架依赖，调用后端API |
| **图标** | Font Awesome 6 | 丰富的UI图标 |
| **HTTP服务** | Python http.server | 本地开发服务器 |

#### 5.3 后端技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **Web框架** | FastAPI | 高性能异步API框架 |
| **Agent框架** | LangGraph | 状态机驱动的Agent编排 |
| **AI集成** | OpenAI兼容API | 支持混元、DeepSeek、Claude等 |
| **数据验证** | Pydantic | 请求/响应模型定义 |
| **服务器** | Uvicorn | ASGI服务器 |

#### 5.5 核心API端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/` | GET | 服务健康检查 |
| `/health` | GET | API健康检查 |
| `/api/diagnosis` | POST | AI账号诊断 |
| `/api/topics` | POST | 智能选题生成 |
| `/api/content` | POST | 内容创作（脚本/标题/文案/封面） |
| `/api/game` | POST | AI小游戏生成 |
| `/api/test` | POST | API连接测试 |

#### 5.6 数据流

```
用户操作 → 前端界面 → HTTP POST /api/game → FastAPI
                                              ↓
                                        content_agent
                                              ↓
                                        ai_client
                                              ↓
                                        OpenAI兼容API
                                              ↓
                                        AI响应 → JSON解析 → 前端展示
```
